Columns 설명¶

데이터 다운 링크에서 train.csv (예측을 위한 학습용 데이터셋)과 test.csv (예측률 확인용 데이터셋)을 다운받아 주세요. 각 데이터셋에서 나오는 칼럼은 아래와 같습니다:

  • Survival - 생존 여부. 0: 사망, 1: 생존
  • Pclass - 티켓 등급. 가장 높은 등급인 1등석부터 가장 낮은 등급인 3등급까지 존재
  • Sex - 성별. 남자(male)와 여자(female)이 있습니다.
  • Age - 나이. 소수점도 존재.
  • SibSp - 같이 탑승한 형재/자매(siblings)와 배우자(spouses)의 총 인원 수.
  • Parch - 같이 탑승한 부모(parents)와 자식(children)의 총 인원 수.
  • Ticket - 티켓 번호.
  • Fare - 운임 요금.
  • Cabin - 객실 번호.
  • Embarked - 선착장. 타이타닉호는 C는 셰르부르(Cherbourg)라는 프랑스 지역, Q는 퀸스타운(Queenstown)이라는 영국 지역, S는 사우스햄튼(Southampton)이라는 영국 지역에서 탑승객을 맞았습니다.

전처리 (Preprocessing)¶

이제 본격적으로 test 데이터의 생존자/사망자를 예측하는 작업을 해보겠습니다!

데이터를 의사결정나무(Decision Tree) 계열의 머신러닝(Machine Learning) 알고리즘에 넣기 위해서는, 데이터를 머신러닝 알고리즘이 이해할 수 있는 형태로 변환해줘야 합니다. 이 과정을 전처리(Preprocessing)라고 합니다.

scikit-learn 에서 제공하는 머신러닝 알고리즘에 데이터를 넣을 수 있는 기본 조건은 다음과 같습니다.

  1. 모든 데이터는 숫자(정수형, 소수점 등)로 구성되어 있어야 한다.
  2. 데이터에 빈 값이 없어야 한다.

여기서는 타이타닉 데이터에 있는 세 컬럼(Sex, Fare, Embarked)을 전처리하는 과정에 대해서 보여드립니다.

Encoding Sex¶

먼저 성별 컬럼을 전처리 합니다.

머신러닝 알고리즘에 넣는 데이터는 모두 숫자형이어야 합니다. 그러니 저희는 성별을 숫자로 바꿔주면 됩니다. 성별 컬럼을 전처리하는 가장 쉬운 방법은 male은 0으로, female은 1로 바꿔주는 것입니다. 이를 전문 용어로 인코딩(Encoding)이라고 합니다.

In [17]:
# 성별(Sex) 값이 male인 경우 0으로, female인 경우 1로 수정합니다.

Encoding Embarked¶

그 다음은 선착장(Embarked)을 전처리해야하는데 성별처럼 0,1,2 이런 식으로 하면.. 사람인 저희는 이해할지라도 decision tree는 헷갈리기 쉽습니다.

예를 들어

  • C == 0
  • S == 1
  • Q == 2 라면 C+S 가 Q 인가? 같은 이상한 계산을 할 수 있습니다.

그러므로 이 경우에는, 1) C, S, Q를 숫자로 변환해주되, 2) 세 값에 특정 연산(덧셈, 뺄셈, 곱셈, 나눗셈 등)을 한다고 동일해지는 현상이 나오지 않도록 인코딩 해줘야 합니다. 이 방식을 원 핫 인코딩(One Hot Encoding)이라고 합니다. 원 핫 인코딩을 해준 선착장은 아래와 같습니다:

  • C == [1, 0, 0]
  • S == [0, 1, 0]
  • Q == [0, 0, 1]

이제 위에서 나온 연산 오류는 안 나오겠죱

파이썬에서는 이것을 엄청 쉽게 boolean 으로 처리할 수 있습니다:

  • C == [True, False, False]
  • S == [False, True, False]
  • Q == [False, False, True]

Training¶

위에서 열심히 전처리 한 데이터를 Decision Tree에 넣어주는 (학습(train)시키는) 과정입니다.

Decision Tree는 머신러닝 알고리즘 중에서 지도학습(Supervised Learning) 이라고 불리우는 알고리즘 중 하나입니다. Supervised Learning 알고리즘을 학습시키기 위해서는 두 가지 타입의 데이터가 필요합니다.

  • Label: 레이블(Label), 내지는 타겟 변수(Target Variable)이라고 부릅니다. (몇몇 곳에서는 Class라고도 부릅니다) 쉽게 말해 우리가 맞춰야 하는 정답입니다. 타이타닉 경진대회에서는 생존 여부(Survived)가 label이 되겠죠.
  • Feature: 우리가 label을 맞추는데 도움이 되는 값들입니다. 타이타닉 데이터에서는 label을 제외한 대부분의 값들이 Feature가 될 수 있습니다.

이번 코드에서 다음의 컬럼을 Feature와 Label로 활용할 것입니다.

  • Feature: 1) 티켓 등급(Pclass), 2) 성별(Sex_encoded), 3) 운임요금(Fare_filled), 그리고 4) 선착장(Embarked)을 사용합니다.
  • Label: 생존 여부(Survived)를 사용합니다.

이를 통해 train 데이터와 test 데이터를 다음의 세 가지 형태의 값으로 나눌 것입니다.

  • X_train: train 데이터의 feature 입니다. 줄여서 X_train이라고 부릅니다.
  • X_test: test 데이터의 feature 입니다. 마찬가지로 줄여서 X_test라고 부릅니다.
  • y_train: train 데이터의 label 입니다. 마찬가지로 줄여서 y_train이라고 부릅니다.

그리고 아시다시피 y_test 라는 데이터는 존재하지 않습니다. test 데이터에는 생존 여부(Survived)를 나타내는 컬럼이 없기 때문이며, 우리의 목표 자체가 y_test에 해당하는 test 데이터의 생존 여부(Survived)를 예측하는 것이기 때문입니다. 그리고 X_train, y_train, X_test를 활용하여 y_test에 해당하는 값을 예측하는 것이 바로 Decision Tree의 역할입니다.

이제 feature와 label을 정의한 뒤, 이를 활용해 X_train, X_test, y_train을 각각 만들어 보겠습니다.

Decision Tree¶

X_train, X_test, y_train 데이터를 만들었으면, 이제 이 데이터를 활용하여 Decision Tree를 학습시켜 보겠습니다. Decision Tree는 머신 러닝 모델 중 하나입니다. 이 외에도 유명한 Random Fores, Logistic Regression 등이 있습니다.

참, 그리고 제가 카페에서 설명드렸던 것과 조금 다릅니다. 그 때는 제가 원래 쓰는 모듈을 사용했는데 괜히 헷갈리실 것 같아 불필요한 부분은 다 뺐습니다.

In [33]:
# scikit-learn(줄여서 sklearn)의 tree 모듈에서 DecisionTreeClassifier를 가져옵니다.
from sklearn.tree import DecisionTreeClassifier

# DecisionTreeClassifier를 생성합니다
# 말씀드린 것 처럼 max_depth 는 너무 과도한 계산까지 안 가도록 하는 것입니다.
decision_tree = DecisionTreeClassifier(max_depth=5)

scikit-learn에 있는 머신러닝 알고리즘에는 크게 두 가지 기능이 있습니다.

  • fit: 머신러닝 알고리즘을 학습시킵니다. 전문용어로 fitting한다고 하기 때문에 fit이라는 표현을 사용합니다. fit을 하기 위해서는 train 데이터가 필요하며, 정확히는 train 데이터의 feature(X_train)와 label(y_train)이 필요합니다. (수학의 함수를 생각하시면 편합니다. (train의) X를 넣어서 Y가 나온 함수가 있는데 (test의) X' 를 넣으면 뭐가 될까? 라고.. 말해주는 것과 같습니다)

  • predict: fit이 끝나면, 이후에 predict를 통해 예측을 할 수 있습니다. predict를 하기 test 데이터의 feature(X_test)가 필요겠죠

결과는 잘 나오긴 한 것 같지만 저게 어떻게 나왔는지는 알 수 없기 때문에 만들어진 트리를 시각화해줄 수 있습니다 (필수 단계 X) Decision Tree를 시각화해서 보기 위해 일반적으로 graphviz 나 webgraphviz.com 를 많이 사용합니다.
